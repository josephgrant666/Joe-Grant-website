Pydantic Models and Schemas

The main piece of learning from this week has been on pydantic models and schemas. These are ways in which code such as response models, classes and database table models can be managed, created and then inherited in path operations. This involves a lot of technical understanding, but once set up, makes the code efficient and easy to maintain.

ORMs - SQLAlchemy

Another big reason for using models and schemas, was to interact with an SQL database directly from the Python app. The aim of this is to eliminate the use of SQL within the program and handle the creation of tables, updates and queries using python only. This also then enables the programmer to handle how responses are sent back to the user. The last main feature of this was authentication and hashing. modules are imported and an algorithm is used to hide information wtihin a table - a vital part of establishing secure data.
